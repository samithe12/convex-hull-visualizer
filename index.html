⚠️ Worst case for Jarvis March: when every point lies on the hull (as in a circle), h = n, so the algorithm does n comparisons for each of n hull vertices — n × n = n² total. With 12 points that is 144 comparisons. Graham Scan always takes only n log₂ n — for 12 points that is roughly 43. This is why algorithm choice matters when datasets are large.
